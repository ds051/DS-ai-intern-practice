实习日志 Day1
 
一、今日任务
 
第一次使用Python程序调用DeepSeek大模型API，借助千问生成调用脚本，实现发送消息“你好”并打印AI返回内容；学习API相关基础概念，掌握API Key安全存储、防泄露规范。
 
二、完整操作步骤
 
1. DeepSeek开放平台准备
访问  platform.deepseek.com  完成账号注册、实名认证，账户充值10元测试额度，在密钥管理页面生成专属API Key，复制保存备用。
2. 借助千问编写调用脚本
向千问输入完整需求，由AI生成DeepSeek接口调用代码；新建 hello_api.py 存放代码；
创建 .env 本地文件存放API密钥，新建 .gitignore 屏蔽密钥文件；
终端执行安装依赖： pip install requests python-dotenv ；
运行  python hello_api.py ，程序成功请求接口并打印AI回复，完整流程跑通。
3. 学习核心专业概念
向通义千问提问，弄懂4个核心知识点并整理记录：API Key含义、messages三种role作用、temperature参数、token计费规则。
4. 落地密钥安全开发规范
了解风险：若把API Key硬编码写进代码并推送到GitHub，密钥会被爬虫窃取，账户充值余额会被盗刷。
解决方案：使用 .env 配置文件存放密钥，代码通过环境变量读取；配置 .gitignore 让Git自动忽略密钥文件；执行 git status 验证， .env 不会出现在待提交列表，杜绝泄露风险。
 
三、核心知识点笔记
 
1. API Key
访问DeepSeek接口的身份凭证，等同于账户密码，持有密钥即可消耗账户额度调用模型，绝对不能明文写在代码、上传公开仓库。
2. messages 三种role
 
- system：系统角色，定义AI身份、回答规则、输出风格；
- user：用户角色，存放人类提问内容；
- assistant：助手角色，存储AI历史回复，维持多轮对话上下文。
 
3. temperature 参数
取值范围0~1，控制模型输出随机性：数值接近0，回答严谨、固定，适合代码/计算；数值接近1，内容发散有创意，适合文案、闲聊。
4. token计费规则
token是大模型文字计量单位，中文约2个汉字折合1个token；计费分为输入token（用户提问）+输出token（AI回复）两部分，双向扣除账户余额。
 
四、安全操作要点
 
1. 严禁将API Key硬编码写入代码并推送GitHub；
2. 使用本地 .env 文件单独存储密钥，代码通过 python-dotenv 读取环境变量；
3. 配置 .gitignore 文件屏蔽密钥配置文件，提交代码时不会上传密钥；
4. 密钥泄露应急处理：登录DeepSeek平台删除旧密钥，重新生成新密钥替换使用。
